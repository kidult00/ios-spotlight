# 项目运行原理

## 核心目标

通过 iPhone 前置 TrueDepth 摄像头实时追踪用户眼睛看向屏幕的哪个位置，并以热力图的形式可视化——看得越多的区域颜色越深。

## 数据流全链路

用户眼睛 → TrueDepth 摄像头 → ARKit 面部追踪 → 3D 眼部数据 → 2D屏幕坐标 → 热图统计 → 热图渲染

分步解释：

1. 硬件层：TrueDepth 摄像头
- iPhone X 及以上机型的前置摄像头，能投射红外点阵捕获面部 3D 结构
- ARKit 的 ARFaceTrackingConfiguration 利用此硬件，以 60fps 实时追踪面部

2. AR 追踪层：ARTrackingManager
- 启动 ARSession，配置面部追踪
- 每帧从 ARKit 收到 ARFaceAnchor，包含：
  - 头部姿态 (transform) — 头在 3D 空间中的位置和旋转
  - 左右眼变换矩阵 (leftEyeTransform / rightEyeTransform) — 眼球相对于头部的位置
  - 注视点 (lookAtPoint) — 用户正在看向的 3D空间点（面部坐标系）

3. 坐标转换层：GazePointConverter（核心算法）

这是最关键的一步——把 3D 空间中的注视方向转为屏幕上的 2D 像素坐标：

- 步骤 A：计算双眼中心点（世界坐标）
  左眼位置 + 右眼位置 → 取平均 → 乘以头部变换矩阵 → 得到眼睛在真实世界中的 3D 位置

- 步骤 B：计算注视方向（世界坐标）
  lookAtPoint（面部坐标系的位置点）→ 乘以头部变换矩阵 → 得到世界坐标中的注视目标
  注视方向 = 注视目标 - 眼睛位置 → 归一化为单位向量

- 步骤 C：射线与虚拟屏幕平面求交
  在摄像头前方约 30cm 处设定一个虚拟平面（模拟手机屏幕）
  从眼睛位置沿注视方向发出一条射线
  计算射线与平面的交点 → 得到 3D 交点

- 步骤 D：3D → 2D 投影
  使用 ARCamera.projectPoint() 将 3D 交点投影到屏幕像素坐标 → 输出 CGPoint(x, y)


4. 热图数据层：HeatmapManager（新建）

  屏幕被划分为 36×64 的网格（每格约 10pt）

  每收到一个注视点坐标：
    → 找到对应的网格位置
    → 以高斯分布在该点周围的网格上叠加权重
      （中心权重最大，向外递减，模拟注视的不确定性）
    → 旧数据随时间衰减（让热图动态变化，而非只增不减）
    → 输出归一化后的强度矩阵 [[Float]]，值域 [0, 1]

5. 热图渲染层：HeatmapOverlayView（新建）

  强度矩阵 → Core Graphics 绘制每个网格点的颜色
    0.0 → 完全透明
    0.2 → 蓝色（低频注视）
    0.5 → 绿色 → 黄色（中频）
    1.0 → 红色（高频注视）

    → 转为 CIImage → 应用 CIGaussianBlur（高斯模糊）
    → 产生平滑柔和的热区效果（而非硬边方块）
    → 渲染为 SwiftUI Image，叠加在摄像头画面上方

6. UI 层：ContentView

    ZStack（从下到上叠放）：
      ├── ARViewContainer  — 摄像头实时画面（RealityKit ARView）
      ├── HeatmapOverlayView — 半透明热图覆盖层
      └── 控制面板 — 重置按钮、显示/隐藏切换
