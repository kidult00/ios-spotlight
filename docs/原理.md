# 项目运行原理

## 核心目标

通过 iPhone 前置 TrueDepth 摄像头实时追踪用户眼睛看向屏幕的哪个位置，并以热力图的形式可视化——看得越多的区域颜色越深。

## 数据流全链路

```
用户眼睛 → TrueDepth 摄像头 → ARKit 面部追踪
    → 3D 眼部数据（眼球旋转矩阵）
    → 注视角度提取（tanX, tanY）
    → 灵敏度放大 / 校准仿射变换
    → 2D 屏幕坐标
    → 眨眼过滤 + EMA 平滑
    → 热图统计（高斯叠加 + 时间衰减）
    → 热图渲染（CIGaussianBlur + 颜色梯度）
```

## 分步解释

### 1. 硬件层：TrueDepth 摄像头

- iPhone X 及以上机型的前置摄像头，能投射红外点阵捕获面部 3D 结构
- ARKit 的 `ARFaceTrackingConfiguration` 利用此硬件，以 60fps 实时追踪面部

### 2. AR 追踪层：ARTrackingManager

- 启动 ARSession，配置面部追踪，30fps 节流处理
- 每帧从 ARKit 收到 `ARFaceAnchor`，包含：
  - **头部姿态** (`transform`) — 头在 3D 空间中的位置和旋转
  - **左右眼变换矩阵** (`leftEyeTransform` / `rightEyeTransform`) — 眼球旋转状态
  - **眨眼系数** (`blendShapes[.eyeBlinkLeft/.eyeBlinkRight]`) — 眼睑闭合程度
- **眨眼检测**：任一眼闭合系数 > 0.5 时冻结注视点，避免眨眼导致数据跳变
- **EMA 指数平滑**：`smoothingFactor = 0.55`（新值权重 45%），约 4 帧延迟，消除帧间抖动
- `autoreleasepool` 包裹帧处理，防止 ARFrame 内存积压

### 3. 坐标转换层：GazePointConverter（核心算法）

将 3D 空间中的注视方向转为屏幕上的 2D 像素坐标：

**步骤 A：提取注视方向**

从左右眼变换矩阵的 Z 轴（`columns.2`）提取注视方向，取平均后归一化。`columns.2` 比 `lookAtPoint` 有更大的角度变化范围。

**步骤 B：转换到摄像头坐标系**

注视方向从面部坐标系 → 世界坐标系（乘以 headTransform）→ 摄像头坐标系（乘以 camera 逆矩阵）。计算注视角度偏移：`tanX = gazeInCam.x / gazeInCam.z`，`tanY = gazeInCam.y / gazeInCam.z`。

**步骤 C：投影到屏幕（两条路径）**

- **默认路径（灵敏度放大）**：tanX/tanY 乘以灵敏度系数（X: 2.5, Y: 3.0），构造摄像头前方 1m 处的合成 3D 点，通过 `ARCamera.projectPoint(.portrait)` 投影到屏幕。灵敏度放大是因为 ARKit 眼球旋转范围（±10-15°）远小于前置摄像头 FOV（~60°），直接映射只能覆盖约 33-50% 屏幕。

- **校准路径（2D 仿射变换）**：用户完成 9 点校准后，通过多元线性回归拟合完整仿射变换 `screenX = ax*tanX + bx*tanY + cx`，直接将注视角度映射到屏幕坐标。使用完整仿射变换（而非独立 X/Y 回归）是因为 portrait 模式下摄像头坐标轴与屏幕坐标轴有 90° 旋转。

### 4. 校准系统：CalibrationManager + CalibrationData

- **9 点校准**：屏幕上依次显示 9 个目标点（10% 边距的 3×3 网格），用户注视每个点 1.5 秒
- **数据处理**：每个点去掉头尾 20% 的过渡帧，取中间 60% 的 tanX/tanY 均值
- **仿射拟合**：正规方程 + Cramer 法则求解 6 个仿射参数
- **持久化**：UserDefaults 存储，App 启动自动加载

### 5. 热图数据层：HeatmapManager

屏幕被划分为约 40×85 的网格（每格约 10pt）：

- **高斯叠加**：每收到一个注视点，以 radius=8, sigma=4.0 的高斯核在周围网格叠加权重
- **时间衰减**：每帧所有网格值乘以 0.97 衰减因子（1 秒后旧值衰减到 40%），使热图反映近期注意力而非全程累积
- **10fps 节流生成热图图像**，后台线程渲染避免阻塞 UI

### 6. 热图渲染层：HeatmapOverlayView

强度矩阵 → Core Graphics 逐像素着色：

```
0.0    → 透明
0~0.25 → 蓝 → 青
0.25~0.5 → 青 → 绿
0.5~0.75 → 绿 → 黄
0.75~1.0 → 黄 → 红
```

→ CIGaussianBlur（sigma=5.0）平滑 → 放大到屏幕尺寸 → SwiftUI Image 叠加显示

### 7. UI 层：ContentView

```
ZStack（从下到上叠放）：
  ├── ARViewContainer     — 摄像头实时画面（RealityKit ARView）
  ├── HeatmapOverlayView  — 半透明热图覆盖层
  ├── 红色注视点指示器     — 20pt 圆形，实时跟随注视位置
  └── 控制面板             — 热图开关、重置按钮、校准按钮（scope 图标）
```

## 关键设计决策

| 决策 | 原因 |
|------|------|
| 灵敏度放大而非直接映射 | ARKit 眼球旋转范围（±12°）远小于摄像头 FOV（±30°），需要 2.5-3x 放大 |
| 完整 2D 仿射而非独立 X/Y 回归 | Portrait 模式下摄像头与屏幕坐标轴有 90° 旋转，需要交叉项 |
| EMA 平滑而非中位数滤波 | EMA 无需缓冲区、O(1) 计算、延迟可控 |
| 时间衰减而非滑动窗口 | 实现简单（每帧乘系数），无需维护时间戳队列 |
| blend shapes 眨眼检测 | ARKit 原生提供，无额外计算开销 |
| 校准数据持久化到 UserDefaults | 数据量极小（6 个 Float），无需数据库 |
